A functional language with delightful features that runs instantly

Open Source and Inspired by Elm, Rust, Rescript, Typescript, OCaml, F#, Python, Ruby, and Clojure

  • Functional language

    • Simple types using Records and Enums
    • - Abstract Data Types made of Record and Enums can model nearly anything, like in Rust, Elm, OCaml and F# 
                        // Record
type Url = {
  scheme : HttpScheme
  domain : String
  port : UInt16
  path : String
  query : Option<String>
}

// Enum (aka Variant, Sum Type, or Abstract Data Types)
type UrlError = 
  | InvalidScheme(String)
  | EmptyDomain
  | InvalidPort(Int64)
  | Unparseable(msg:String, context:String)

// Aliases are just shorthands
type UrlParseResult = Result<Url, UrlError>
    • - We believe Object Oriented programming is a terrible way to model programs, and inheritence is a terrible misfeature. Languages like OCaml, Elm, and F# have shown that nearly all problems can be modeled using ADTs.
  • Runs Instantly

    • Instantly run any package from the CLI: darklang @username.functionName arg1 arg2

    • Streaming package manager with no npm install step

    • Single binary to install - no environments or containers to setup

    • Interpreted with no compile step
  • Delightful features

    • Works with VS Code (or any LSP-editor) and GitHub Copilot

    • Gradual Static Typing

      • When prototyping, don't let the type checker hold you up - just run your program until you hit an error

      • 2024 After prototyping, run the full type checker to gain confidence your whole program works

    • Optional 2024 Instant cloud deployment of code (to our cloud or yours), with instant creation of DBs, API endpoints, and worker queues, with no containers, no cold starts, no orchestration, or other devops/cloud engineering required

    • Package items (functions, types and constants) are versioned and immutable, taking the worry out of taking on dependencies

      • Only download the specific package items you use

      • Only upgrade the specific package items you use

      • 2024 Automated dependency upgrades, as we track deprecation status, and know what functions are pure and safe to update.

      • Different packages can rely on different versions of other packages

      • Use multiple versions of the same package item at once: allows testing new versions without having to change an entire package version, lowering risk.

      • 2024 Share pre-release functions trivially, without contributors needing to check out your git repo or set up anything

    • -

      2024 The package manager functions as a source repository - no need for uploads, releases or other synchronization. No git or GitHub required (but you can sync to GitHub if you prefer).
  • Designed for GenAI

    • Works with GitHub Copilot

    • We redesigned the dark language and tooling to enable GenAI-generated programs, including exposing language tools to GenAI tools, allowing running partial and incomplete programs safely, and ensuring access to significant context to GenAI tools

    • 2024 Build short CLI programs from prompts darklang prompt "find all js files which don't have a CSS file of the same name"

    • 2024 Build vendor SDKs from prompts and OpenAPI docs

    • 2024 Build complex programs with darklang AI agents
  • Open source
    • We expect to fully open source Darklang, which is currently source available, in 2023 or early 2024
