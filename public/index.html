A functional language with delightful features that runs instantly

Open Source and Inspired by Elm, Rust, Rescript, Typescript, OCaml, F#, Python, Ruby, and Clojure

At a glance

  • Modern functional language

    • -

      Simple and powerful runtime: static type system with simple immutable types, composition via records and enums, Option and Result types, garbage collection, automatic async 2024, and Unicode strings/chars. No null, exceptions, async/await, or complex type rules

    • -

      Simple setup: single static binary, with no environments or containers to setup. Streaming package manager means no packages to install. Interpreted execution means no compile step

    • -

      Fast prototyping with instant startup (no compile step), error shortcuts 2024, and gradual static typing (just run programs until they hit an error)

    • -

      2024 Full type checking: after prototyping, run the static typechecker to catch errors - and get confidence your program works

  • Streaming Package manager
    • - Our package manager streams functions, types and constants as they're used, so there's no package install step, and no installing packages before you can use them
    • - Package features (constants/functions/types) are individually versioned and immutable:
      • - Fewer dependencies: no package versions so no need to upgrade unless the functions you use have newer versions
      • - Use multiple versions of the same function at once: allows testing of new dependency versions with less risk using controlled rollout

      • -

        2024 Pre-release functions trivially, without users needing to check out your git repo or set up anything

      • -

        2024 We track deprecation status, and know what functions are pure/safe to update, so we can automate most remaining dependency management automatically

    • - The package manager is the source repository - no more releases, uploads, and synchronization. No git or github required (but feel free, if you prefer). Less friction for contributors
  • Run from anywhere
    • - CLI runs any packages with no install: darklang @username.functionName arg1 arg2 from the command line (or the browser, Slack, Discord, VS Code, etc)
    • - Optional 2024 Instant cloud deployment of code (to our cloud or yours), with instant creation of DBs, API endpoints, and worker queues, with no containers, no cold starts, no orchestration, or other devops/cloud engineering required
  • Redesigned for GenAI
    • - Works with GitHub Copilot
    • - We redesigned the dark language and tooling to enable GenAI-generated programs, including exposing language tools to GenAI tools, allowing running partial and incomplete programs safely, and ensuring access to significant context to GenAI tools
    • -

      2024 Build short CLI programs from prompts darklang prompt "find all js files which don't have a CSS file of the same name"

    • - 2024 Build vendor SDKs from prompts and OpenAPI docs
    • - 2024 Build complex programs with darklang AI agents
  • Open source
    • In 2023, we expect to fully open source Darklang, which is currently source available
